Show me the API

BinaryJS allows multiple streams over a single realtime websocket connection. Either the client or server can create a stream and both client and server can write to it once created.


var server = BinaryServer({port: 9000});
server.on('connection', function(client){
  client.on('stream', function(stream){
    var w = fs.createWriteStream(stream.id+'.txt');
    stream.pipe(w);
  }); 
});

Binary data for the web is here

A year ago if someone asked you "how do I stream binary data / audio / video / files to Javascript", the answer was "Flash" or "no" (or "Java applets")

But as of Chrome 15+, Firefox 11+, Internet Explorer 10, and Safari nightly builds, that is no longer true.

BinaryJS is a light but comprehensive framework utilizing websockets to:

  • Send, stream, or pipe binary data between web javascript and Node.js
  • Data is never stringified. Everything is only ever binary
  • BinaryPack serialization is included and fully JSON compatible
  • Multiplexed websocket streams. A client and server can send any number of streams to each other
  • Adherence to Node.js read/write Streams API so you can pipe any stream into BinaryJS streams (and vice-versa)
  • "pause", "resume", and "end" streams as in the Streams API

What can you do with this?

You have just as much flexibility as a full tcp socket between client and server

Here are some ideas that are now possible (beta-testers are already working on some!):

  • Streaming FLAC into HTML5 WebAudio api
  • Progressive image loading (for retina displays and high-res photorgraphy)
  • Multiplayer video games without expensive stringifying
  • Streaming file uploads (see example)
  • Live HTML <video> streaming
  • PJAX with no http requests. Not even for image assets.
  • Video / audio chat over websockets
